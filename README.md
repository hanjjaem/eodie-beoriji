# 어디에 버리지? — 부산 동구 생활배출 가이드

품목명을 입력하면 **음식물 / 일반 / 별도처리** 중 어디에 버려야 하는지, **왜** 그렇게 분류되는지, **어떻게** 버리는지를 즉시 알려주는 다국어 분리배출 도우미입니다.

부산 동구 주민(특히 한국 분리배출 규칙에 익숙하지 않은 외국인 주민)을 주요 대상으로 만들어졌습니다.

> 🔗 **라이브 데모:** https://hanjjaem.github.io/eodie-beoriji/

## ✨ 주요 기능

- **품목 검색** — 달걀껍데기, 수박, 생선뼈처럼 품목명을 입력하면 분류 결과를 바로 표시
- **퍼지 매칭** — 완전일치 → 부분일치 → 단어 포함 순으로 점수를 매겨 오타·유사어도 최대 3개까지 추천
- **분류 이유 설명** — 뼈·껍질·씨앗·염분·기름 등 카테고리별로 "왜 이렇게 버리는지" 안내
- **배출 방법 단계** — 수분 제거, 봉투 선택 등 실제 배출 절차 제공
- **4개 언어** — 한국어 · English · 中文 · Tiếng Việt (언어별 검색어 정규화 포함)
- **반응형 디자인** — 모바일 우선, 데스크톱까지 대응
- **동별 미수거 문의 연락처** — 초량·수정·좌천·범일동 담당 수거업체 전화 연결

## 🗂 프로젝트 구조

```
.
├── index.html     # 마크업 (검색 UI · 결과 모달 · 연락처)
├── styles.css     # 전체 스타일 (반응형 브레이크포인트 포함)
├── data.js        # 콘텐츠/번역 레이어 — 품목 DB(LANGS), UI 문구(COPY), 분류 메타(META)
└── app.js         # 동작 레이어 — 검색 매칭, 결과 렌더링, 언어 전환, 모달
```

`data.js` 가 먼저, `app.js` 가 나중에 로드됩니다(전역 스코프 공유). 빌드 도구·의존성 없이 정적 파일만으로 동작합니다.

## 🚀 로컬에서 실행

별도 빌드가 필요 없습니다. 다음 중 하나면 됩니다.

```bash
# 1) 그냥 파일 열기
#    index.html 더블클릭

# 2) 간단한 로컬 서버 (권장 — file:// 제약 없이 동작)
npx serve .
# 또는
python -m http.server 8000
```

브라우저에서 `http://localhost:8000` 접속.

## 🧩 데이터 추가/수정 방법

품목을 추가하려면 `data.js` 의 각 언어 `db` 배열에 항목을 추가합니다.

```js
{
  t: ["검색 키워드", "동의어1", "동의어2"], // 매칭에 쓰이는 키워드들
  c: "food",                               // food | general | special
  d: "화면에 표시될 품목 설명",
  s: ["배출 방법 1단계", "2단계", ...]       // 단계별 안내
}
```

- `c` 값에 따라 결과 색상·아이콘·봉투 안내가 결정됩니다 (`META`, `FALLBACK_BAG` 참고).
- 새 품목은 **4개 언어 모두에 추가**해야 모든 언어에서 검색됩니다.

## 🌐 분류 기준

| 카테고리 | 의미 | 예시 |
|---|---|---|
| 🍚 `food` (음식물) | 처리시설에서 재활용 가능한 부드러운 음식 잔재물 | 과일껍질, 채소 자투리, 잔반 |
| 🗑️ `general` (일반) | 뼈·껍질·씨앗처럼 음식물 재활용에 부적합한 것 | 달걀껍데기, 큰 뼈, 조개껍데기, 과일씨 |
| ⚠️ `special` (별도처리) | 흡수·전용 수거가 필요한 것 | 폐식용유 |

## ⚠️ 면책

분류 기준은 부산 동구 기준 일반 안내이며, 실제 배출 규정은 지자체·시점에 따라 달라질 수 있습니다. 정확한 사항은 관할 구청 또는 동별 수거업체에 확인하세요.
